# Documentation Standards

This directory contains the institutional-control layer of the repository. Its role is to define measurement boundaries, inference rules, attribution discipline, reproducibility standards, and publication constraints before any substantive conclusion is drawn from the public data.

## Files

- `startup-data-governance-and-source-map.md` — source hierarchy, ownership boundaries, and repository measurement logic.
- `bay-area-geographic-scope-and-fips-governance.md` — Bay Area geographic definition and FIPS governance.
- `startup-data-dictionary-and-measurement-boundaries.md` — variable dictionary and interpretation limits.
- `model-governance-and-inference-boundaries.md` — acceptable and unacceptable claims.
- `reproducibility-protocol-and-citation-standard.md` — rerun discipline, citation logic, and provenance standards.
- `identification-robustness-and-sensitivity-standard.md` — robustness hierarchy and sensitivity conventions.
- `nonpublic-data-handling-and-attribution-protocol.md` — source-handling and publication boundaries.
- `feature-engineering-and-alert-threshold-standard.md` — construction rules for derived indicators and alert thresholds.
- `public-release-and-source-attribution-standard.md` — source-control and public-release requirements.
- `out-of-time-validation-and-backtesting-standard.md` — real-time validation and rolling historical evaluation standards.
- `uncertainty-decomposition-and-confidence-band-standard.md` — uncertainty decomposition and confidence-band rules.
- `repository-integration-and-validation-note.md` — repository-level validation protocol.
- `repository-map.md` — full repository inventory and file-role map.

## Use

Read this directory before extending the data model, adding new inference claims, publishing charts or derived indicators, or porting the repository to another regional ecosystem.
