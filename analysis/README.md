# Analytical Papers

This directory contains the repository’s substantive research notes. Each document is self-contained, but together they form a coordinated analytical system spanning entry dynamics, employer-conversion proxies, concentration, near-term scenario design, hazard-style diagnostics, backtesting, and county resilience assessment.

## What this layer produces

The analytical layer translates the repository’s processed evidence base into:
- county panels and structural patterns,
- concentration metrics and scaling asymmetries,
- employer-conversion proxy logic,
- near-term scenario bands,
- hazard diagnostics and regime-shift signals,
- resilience rankings and monitoring outputs.

## Files

- `startup-survival-nowcasting-empirical-design.md` — empirical design for Bay Area startup-pipeline nowcasting.
- `bay-area-business-applications-structural-patterns.md` — county application structure, scale, and concentration.
- `bay-area-employer-formation-bridge-model.md` — county bridge model from applications to employer-forming startup proxies.
- `bay-area-county-concentration-and-scaling-patterns.md` — concentration asymmetries and county dependence.
- `bay-area-startup-formation-cycle-2005-2024.md` — long-run formation cycle and post-shock normalization.
- `bay-area-near-term-startup-scenario-framework.md` — near-term scenario design for startup-pipeline interpretation.
- `bay-area-startup-hazard-proxy-system.md` — hazard-style monitoring system using public indicators.
- `bay-area-county-turning-point-and-regime-shift-diagnostics.md` — turning points, structural breaks, and ranking reversals.
- `bay-area-real-time-nowcast-backtesting-framework.md` — pseudo-real-time evaluation of annual nowcasts.
- `bay-area-county-resilience-scorecard-and-monitoring-system.md` — county resilience, volatility, and monitoring scorecard.

## Use

These papers are intended for academic reference, technical reading, investment-screening overlays, and policy-oriented ecosystem analysis. They should be read together with the control standards in `docs/` and the acquisition and reproducibility rules defined in the repository root.
